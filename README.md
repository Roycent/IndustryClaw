# IndustryClaw

班组长工作台版本。

这是一个围绕**班组长单角色**重构的工业现场产品原型。界面目标很明确：用户一打开就知道这是给班组长看的，先处理什么，下一步该点哪里。

基于 React + Vite + TypeScript，当前版本不再做“多角色平均分配”的首页，而是把 4 个视图全部收敛到班组长工作流：

1. **首页 / 当班总览**
   - 当班总览
   - 高优先级异常
   - 待分派任务
   - 未回执人员
   - 即将超时
   - 交接待确认

2. **设备助手**
   - 班组长提问设备异常
   - 返回原因摘要
   - 返回当前责任人
   - 返回建议动作
   - 判断是否生成工单 / 催办 / 加入交接 / 升级

3. **工单中心**
   - 只看谁接单了
   - 谁没接
   - 谁超时
   - 哪个要升级
   - 哪个要进交接

4. **交接页**
   - 本班异常
   - 未结项
   - 接班责任人
   - 接收确认

## 文案方向

全部围绕班组长真实工作语言：

- 待分派
- 待回执
- 超时
- 升级
- 接班人
- 交接确认
- 风险
- 影响产线
- 是否停机
- 是否升级

保留了设备详情、工单详情、SOP 引用等内容，但都只为班组长决策服务，不展开成其他岗位的完整工作台。

## 技术栈

- Vite
- React 18
- TypeScript
- CSS

## 本地运行

```bash
npm install
npm run dev
```

默认地址：

```bash
http://localhost:5173
```

## 构建

```bash
npm run build
```

构建产物输出到：

```bash
dist/
```

## 部署

推荐静态部署到 Vercel。

建议配置：

- Framework Preset: `Vite`
- Build Command: `npm run build`
- Output Directory: `dist`
