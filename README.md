# IndustryClaw

> 面向工业现场班组长与值班团队的多智能体工作台。

**GitHub About / 仓库简介短文案**

IndustryClaw 是一个面向制造业夜班值守、设备异常处置、工单催办与交接编排的工业多智能体产品原型。它不是单一聊天机器人，而是由 10 个现场分工明确的 agents 与 30 个细颗粒度 skills 组成的值班协同系统。

## 项目定位

IndustryClaw 不是“给工厂套一个聊天框”，而是把班组长夜班真正会盯的事情做成一个像生产系统一样可操作的工作台：

- 先看哪条线被卡住
- 谁还没接、谁还没回
- 哪个异常应该升级
- 哪些事项必须带班交给白班
- SOP、故障经验、停机口径、影响范围如何快速补齐

产品页面保持工业现场值班风格，强调“盯责任、盯卡点、盯影响、盯交接”，而不是解释性 PPT 页面。

## 10 个 agents 概述

IndustryClaw 采用现场多智能体体系，10 个 agents 按岗位与场景拆分：

1. **班组长助手**：值班总控，拉优先级、拍升级口径、统一班次决策
2. **设备异常分析**：针对设备重复短停、异常波动做现场诊断
3. **工单协调**：盯未接单、未回执、超时催办与责任补派
4. **交接整理**：把未闭环和未稳设备整理成白班先看清单
5. **SOP 检索**：快速匹配作业指导、点检路径、应急步骤
6. **能耗异常**：面向 BAC、空压、冷站等公辅系统做偏差识别
7. **质量波动**：识别工艺漂移、批次风险、质量扩散可能性
8. **备件建议**：判断能否先换、换什么、库存是否可替代
9. **停机归因**：把短停/停机的归因与责任归口说清楚
10. **升级判断**：根据停机频次、超时与影响范围决定是否升级

这些 agents 不是平铺列表，而是自然嵌在首页、设备助手、工单中心、交接页与调度台中：

- 首页展示“当前编排 agents / 当前落地 skills”
- 设备助手展示“当前调用 agents / skills”
- 工单中心展示每张工单的主责 agent 与支撑 skills
- 交接页展示每个交接事项由哪个 agent 收口
- 新增“多智能体调度”视图，集中展示 10 个 agents 的现场分工与本班调用情况

## 30 个 skills 概述

30 个 skills 按现场业务组织成 3 组，而不是做成一大串说明文档：

### 1) 班次与工单协同（8 个）

- 班次优先级排序
- 催回执
- 责任人校验
- 超时盯防
- 交接摘要
- 白班接收检查
- 闭环缺口识别
- 带班口径生成

### 2) 设备诊断与现场处置（8 个）

- 停机归因
- 故障案例检索
- 信号关联比对
- SOP 匹配
- 点检路径推荐
- 应急步骤提示
- 短停分类
- 停机责任归口

### 3) 能耗、质量与备件判断（14 个）

- 能耗基线比对
- 停机窗口建议
- 负荷漂移识别
- 质量波动识别
- 批次影响评估
- 关键参数守卫
- 备件校验
- 寿命剩余评估
- 替代备件建议
- 影响产线评估
- 责任链识别
- 升级责任链追踪
- 异常升级阈值判断
- 影响线体范围判断

这些 skills 在界面里的体现方式是：

- 跟随具体设备、工单、异常、交接事项显示“当前调用 skills”
- 在调度台里按模块展示 skill 组
- 每个 agent 卡片展示自己常用的 skills 组合

## 适用场景

IndustryClaw 适合以下工业场景：

- 制造业夜班值守 / 班组长当班盯防
- 包装线、装配线、成型线等离散制造现场
- 公辅设备异常（冷站、BAC、空压站、循环系统）
- 工单催办、回执追踪、升级判断
- 班次交接、白班接收、未闭环事项承接
- 需要把 SOP、案例、责任链、停机口径和现场动作串起来的场景

## 当前界面重点

保留并强化 5 个视图：

- **首页**：值班先看什么，以及当前调用了哪些 agents / skills
- **设备助手**：右侧详情加入当前 agent 与 skill 面板，直接盯卡点、责任、升级口径
- **工单中心**：每张工单都能看到主责 agent，突出未接、未回、超时、升级、带班交
- **交接页**：白班先看、未稳设备、未闭环，并明确交接事项由哪个 agent 收口
- **多智能体调度**：集中展示 10 个 agents 与 30 个 skills 的体系化编排
