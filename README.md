# IndustryClaw

工业现场任务协同产品原型。

基于 React + Vite + TypeScript，界面定位为深色工业风的现场执行系统原型，围绕班次值守、工单执行、告警处置、交接台账、设备助手工作台展开。

## 页面结构

### 1. 生产总览

包含：

- 班次统计卡
- 当班待办
- 值守状态表
- 设备列表 + 右侧设备详情面板
- SOP 文档编号 / 版本 / 适用设备 / 最近引用时间
- 关键检查项与故障案例面板
- 现场记录采样表

### 2. 设备助手工作台

包含：

- 自然语言查询入口对应的执行链视图
- 助手输入 / 输出记录
- SOP / 故障案例引用详情侧栏
- 写入结果面板
- 回执追踪表
- 当前任务项与截止时间

### 3. 告警 / 工单中心

包含：

- 工单执行列表
- 工单详情面板
- 状态流转步骤
- 处理意见记录
- 升级记录
- 告警列表
- 批量派发 / 催办 / 升级 / 导出交接项按钮

工单详情字段已补齐：

- 工单号
- 来源
- 优先级
- 责任人
- 计划开始
- 截止时间
- 当前节点
- 处理意见
- 升级记录

### 4. 班组交接 / 任务历史

包含：

- 交接记录头信息
- 交接签收单列表
- 交接详情 / 签收动作区
- 接收确认表
- 未结项表
- 附件 / 备注占位
- 异常时间线
- 岗位回执矩阵

交接详情字段已补齐：

- 交接单号
- 交出 / 接收
- 接收状态
- 签收时间
- 未结项
- 附件占位
- 备注

## 界面方向

当前版本继续强化为更像真实系统的界面：

- 维持深色、克制的工业软件风
- 减少解释型文案
- 增加详情卡、字段组、状态标签、步骤状态、侧边详情区
- 页面重点转为“列表 + 明细 + 回执 + 升级 + 引用记录”的真实产品结构

## 技术栈

- Vite
- React 18
- TypeScript
- CSS

## 本地运行

```bash
npm install
npm run dev
```

默认地址：

```bash
http://localhost:5173
```

## 构建

```bash
npm run build
```

构建产物输出到：

```bash
dist/
```

## 部署

推荐静态部署到 Vercel。

建议配置：

- Framework Preset: `Vite`
- Build Command: `npm run build`
- Output Directory: `dist`

`vite.config.ts` 已配置 `base: './'`，可用于常规静态托管或子目录部署。
